# Trust Badge — local proof viewer

A small static, offline proof viewer for the Mandate Open Agents demo. It is
not a marketing landing page. It is a one-screen operational summary of the
proof points the demo just produced, suitable for a judge skimming on a
laptop.

## What it shows

Side-by-side allow / deny scenarios from the most recent demo run, plus the
no-key proof and the audit-chain tamper detection results:

- Agent identity: `agent_id`, `demo_commit`, `generated_at_iso`,
  `schema: "mandate-demo-summary-v1"`.
- **Allow (`legit-x402`)**: `decision`, `matched_rule`, `request_hash`,
  `policy_hash`, `audit_event`, `receipt_signature`,
  `keeperhub_execution_ref` with an explicit `mock` tag.
- **Deny (`prompt-injection`)**: `decision`, `deny_code`, `matched_rule`,
  `request_hash`, `policy_hash`, `audit_event`, `receipt_signature`,
  `denied_action_executed: false`, `keeperhub_refused: true`.
- **No-key proof**: `status` (PASS / FAIL) plus the three falsifiable counts
  (`agent_source_signer_references`, `agent_cargo_signer_deps`,
  `agent_key_material_files`).
- **Audit chain**: `structural_verify_accepts_tampered_actor`,
  `strict_hash_verify_rejects_tampered`.
- Footer reminder that KeeperHub / Uniswap executors are local mocks and
  ENS uses an offline resolver fixture.

Tagline at the top: **"Don't give your agent a wallet. Give it a mandate."**

## How to use

From the repo root:

```bash
# 1. Run the demo. Step 13 writes the deterministic JSON used as input.
bash demo-scripts/run-openagents-final.sh

# 2. Render the viewer.
python3 trust-badge/build.py

# 3. Open it. Pick the command for your OS.
open trust-badge/index.html        # macOS
xdg-open trust-badge/index.html    # Linux
start trust-badge/index.html       # Windows
```

The HTML is self-contained: no JS, no external CSS, no external fonts, no
network calls. It works directly from `file://` — no local web server
needed.

## Why a generator instead of static HTML + `fetch()`

Browsers block `fetch('latest-demo-summary.json')` when the page is loaded
from `file://` (same-origin policy). A fetch-based viewer would silently
fail when a judge double-clicks the file. `build.py` instead pre-renders
every value into the HTML, so the file works offline with no setup.

## CLI

```text
python3 trust-badge/build.py [--input PATH] [--output PATH]

  --input    Demo summary JSON
             (default: demo-scripts/artifacts/latest-demo-summary.json)
  --output   Static HTML viewer
             (default: trust-badge/index.html)
```

The script refuses to render if the input's `"schema"` field is anything
other than `mandate-demo-summary-v1`. This protects against silent drift if
the JSON contract changes in a future PR.

## Honest scope

- **Mocks remain mocks.** The viewer renders KeeperHub / Uniswap executors
  with a `mock` tag and reproduces the demo's "denied actions never reach
  the sponsor" claim from the captured `keeperhub_refused: true`. It does
  not invent or hide truthfulness fields.
- **Values are real, not curated.** Every field is rendered straight from
  the JSON the demo runner produced; the viewer adds no interpretive
  marketing copy.
- **Verifies, does not validate.** This viewer summarises proof points the
  demo runner already verified. Re-running `run-openagents-final.sh` is the
  authoritative re-verification path — the viewer is a presentation layer
  on top.
