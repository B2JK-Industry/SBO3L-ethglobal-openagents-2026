# v1.2.0 release prep — Phase 2 closeout

> **Audience:** QA + Release (Heidi) at Phase 2 close.
> **Outcome:** v1.2.0 published cleanly across crates.io / npm / PyPI in under 60 minutes from "Phase 2 done" call.
> **Pattern:** mirrors the v1.0.1 PR (#156) shape exactly — bump → tag → workflow-driven publish.

This is a **runbook**, not a changelog draft. The actual `CHANGELOG.md` v1.2.0 section gets filled in immediately before tag day, when the merged-PR list is final.

## Trigger conditions

v1.2.0 is the Phase 2 closeout release. Cut it when **all** of the following are true:

- Track 1 (KH frameworks): 8/8 tickets ✅ (already met as of v1.0.1)
- Track 3 (ENS Most Creative): T-3-1..T-3-7 all merged
- Track 4 (ENS AI Agents): T-4-1..T-4-3 all merged (including ERC-8004)
- Track 5 (Uniswap Best API): at least 50% of T-5-1..T-5-6 merged (T-5-1, T-5-5, T-5-6 minimum)
- CTI-3: marketing main + docs main + hosted preview slice 2 deployed
- Phase 2 exit gate test commands (`docs/win-backlog/08-exit-gates.md` Phase 2 section) all green
- Daniel signs off

If any criterion is amber, hold the tag — partial publishes are worse than late publishes.

## Pre-tag verification (Heidi runs)

```bash
# 0. Sync, branch
git checkout main && git pull --ff-only origin main
git checkout -b chore/v1.2.0-release

# 1. Full regression sweep (or rely on regression-on-main.yml's last run)
cargo fmt --all -- --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace --all-targets
python3 demo-fixtures/test_fixtures.py
python3 trust-badge/test_build.py
python3 operator-console/build.py --input operator-console/fixtures/operator-summary.json --evidence operator-console/fixtures/operator-evidence.json
python3 operator-console/test_build.py
bash demo-scripts/run-openagents-final.sh
bash demo-scripts/run-production-shaped-mock.sh

# 2. cargo publish --dry-run on each crate (catches metadata gaps pre-tag)
for crate in sbo3l-core sbo3l-storage sbo3l-policy sbo3l-identity sbo3l-execution \
             sbo3l-keeperhub-adapter sbo3l-server sbo3l-mcp sbo3l-cli; do
  echo "=== $crate ==="
  cargo publish -p "$crate" --dry-run 2>&1 | tail -10
done
# Any "error: failed to verify" → fix before bumping versions

# 3. Same for SDKs
cd sdks/typescript && npm pack --dry-run | tail -20 && cd -
cd sdks/python && uv build --no-isolation && twine check dist/* && cd -
```

If any dry-run fails, file an issue on the failing crate's owner and abort the release.

## Version bump

Mirror PR #156's shape:

1. `cargo set-version --workspace 1.2.0` (uses `cargo-edit`; if not installed, hand-edit `Cargo.toml`s)
2. Update internal-dep version constraints in workspace `Cargo.toml`:
   ```toml
   sbo3l-core = { version = "1.2.0", path = "crates/sbo3l-core" }
   # ... and the other 8 internal deps
   ```
3. Update `sdks/typescript/package.json` → `"version": "1.2.0"`
4. Update `sdks/python/pyproject.toml` → `version = "1.2.0"`
5. Update each framework integration package's version:
   - `integrations/langchain-typescript/package.json`
   - `integrations/autogen/package.json`
   - `integrations/elizaos/package.json`
   - `integrations/vercel-ai/package.json`
   - `integrations/langchain-python/pyproject.toml`
   - `integrations/crewai/pyproject.toml`
   - `integrations/llamaindex/pyproject.toml`
   - `integrations/langgraph/pyproject.toml`
6. Widen Python integrations' `sbo3l-sdk` constraint to `>=1.2.0,<2`
7. Update `packages/design-tokens/package.json` if its public surface changed

## Tag flow

```bash
# All bumps committed to chore/v1.2.0-release branch
git add -A && git commit -m "chore(release): v1.2.0 — Phase 2 closeout"
git push -u origin chore/v1.2.0-release

# Open PR (mirrors #156 body shape)
gh pr create --base main --head chore/v1.2.0-release \
  --title "chore(release): v1.2.0 — Phase 2 closeout" \
  --body-file docs/release/v1.2.0-pr-body.md   # template below

# After merge:
git checkout main && git pull --ff-only origin main
git tag -a v1.2.0 -m "Phase 2 — ENS depth + Uniswap depth + frameworks"
git tag -a sdk-ts-v1.2.0 -m "TS SDK v1.2.0"
git tag -a sdk-py-v1.2.0 -m "Py SDK v1.2.0"
git push origin v1.2.0 sdk-ts-v1.2.0 sdk-py-v1.2.0
```

## Post-publish smoke (Heidi runs)

```bash
# 1. Fresh-env smoke against 1.2.0 versions
rm -rf /tmp/sbo3l-v1.2-smoke && mkdir /tmp/sbo3l-v1.2-smoke && cd /tmp/sbo3l-v1.2-smoke
cargo init --bin --name sbo3l-v1.2-smoke
# add all 9 crates at "1.2.0" to Cargo.toml + a small main.rs exercising:
#   - request_hash
#   - Storage::open_in_memory
#   - sbo3l-cli installs + executes (`cargo install sbo3l-cli --version 1.2.0 && sbo3l --version`)
cargo build --release && ./target/release/sbo3l-v1.2-smoke

# 2. SDK smokes
mkdir /tmp/sbo3l-sdk-v1.2-smoke && cd /tmp/sbo3l-sdk-v1.2-smoke
npm init -y && npm install @sbo3l/sdk@1.2.0
node -e "const sdk = require('@sbo3l/sdk'); console.log(sdk.VERSION);"
# expect: 1.2.0

mkdir /tmp/sbo3l-py-v1.2-smoke && cd /tmp/sbo3l-py-v1.2-smoke
python3 -m venv venv && source venv/bin/activate
pip install sbo3l-sdk==1.2.0
python3 -c "import sbo3l_sdk; print(sbo3l_sdk.__version__)"
# expect: 1.2.0
```

If any of those fail, **revert by yanking** (don't delete) the broken crate version on crates.io / `npm deprecate` the npm version / `pip` no-op (PyPI can't unpublish) and ship a v1.2.1.

## Rollback / hotfix decision tree

```
v1.2.0 published, regression detected:
├── Critical (broken install / signing wrong / data corruption):
│   ├── Yank broken crate version (does NOT remove deps that already pulled)
│   ├── npm deprecate <version> --message "use 1.2.1"
│   ├── PyPI: contact admin (cannot unpublish; ship 1.2.1 fast)
│   └── Open hot-fix PR → 1.2.1 within 24h
└── Non-critical (cosmetic / docs):
    └── Roll into next release; note in CHANGELOG
```
