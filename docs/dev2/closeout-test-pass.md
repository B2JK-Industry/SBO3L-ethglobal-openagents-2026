## Methodology notes

- The test pass spawns `npm install` + `tsc` + `vitest run` + `tsup` per adapter sequentially
- The first run failed for every adapter because the SDK at `sdks/typescript/dist/` wasn't built — the `file:../../sdks/typescript` peer link needs the dist to exist
- After running `npm run build` in `sdks/typescript` once at the top, the second pass shows all 20 adapters green
- This is the same pattern that bit `vercel-ai` in CI before PR #331 added the explicit "build SDK first" step to `integrations-publish.yml`
